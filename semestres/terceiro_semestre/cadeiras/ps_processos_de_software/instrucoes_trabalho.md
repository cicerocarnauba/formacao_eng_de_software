# Instruções do Trabalho

## **Objetivo**

Permitir que os alunos compreendam, apliquem e analisem técnicas de modelagem de processos de negócio utilizando a notação **BPMN**, aproximando-os da prática real de análise e melhoria de processos em organizações.

Para alcançar esse objetivo, os alunos deverão **selecionar uma área do desenvolvimento de software de uma organização real** (exemplo: NPI), analisá-la e modelá-la com BPMN e propor eventuais melhorias. O trabalho será realizado em **grupos de 8 integrantes**.

## **Etapas**

- **Etapa preliminar: Seleção da Área e Organização**
    
    Cada grupo deverá escolher uma área do desenvolvimento de software de uma organização (exemplo: NPI ou qualquer outra organização/empresa/instituição de desenvolvimento de software). As seguintes áreas devem ser escolhidas:
    
    - Requisitos
    - User Experience (OBS: NPI não tem essa área)
    - Desenvolvimento (front end..)
    - CI/CD (Integração Contínua e Entrega Contínua) - devops ruben coordenador
    - Testes de Software (OBS: NPI não tem essa área)
    - Correção de bugs (OBS: NPI não tem fluxo de trabalho bem definido para essa área)
    - Gerenciamento de projeto
        - Estimativas
        - Planejamentos
    
    P.S.: Pode ter grupos com as mesmas área, porém devem ser de organizações diferentes.
    
    **Entregável:** mensagem no *discord* com a área e organização escolhida pelo grupo.
    
    prazo: essa sexta.
    
- **ETAPA 1. Levantamento, Modelagem e Validação do Processo Atual - “AS IS”**
    
    O grupo deverá:
    
    - Realizar entrevistas para obter dados/informações sobre fluxo de trabalho atual realizado para a área que você escolheu.
        - Identificar participantes (atores ou departamentos) do fluxo de trabalho: quais são as pessoas e/ou departamentos envolvidos?
        - Identificar atividades, tomadas de decisão, resultados das atividades: qual é o passo a passo realizado para alcançar um determinado objetivo?
    - Descrever o conjunto de atividades realizadas na área escolhida da organização
    
    *P.S.: Caso o grupo escolha o NPI, os melhores dias para abordá-los são Quinta e Sexta.*
    
    - Criar o diagrama BPMN com base nos dados acima contendo: **Pools, Lanes, Eventos, Tarefas, Gateways, Fluxos, entre outros.**
    
    **Entregável: documento com descrição das atividades (seguir o modelo em [Modelos para Especificação de (Sub) Processos](https://www.notion.so/Modelos-para-Especifica-o-de-Sub-Processos-271fc0081bb6810aaa3af2953080269d?pvs=21)) e** apresentação descrevendo detalhadamente:
    
    - Organização (objetivos (o que ela faz?), valores, como está organizada, etc)
    - Obtenção de dados (entrevista? presencial? com quem? onde? como? trazer evidências ex. foto, ata, etc)
    - Fluxo de trabalho atual na área (conjunto de atividades realizadas)
    - Diagrama BPMN “AS IS”
    - Validação (como foi realizada? trazer evidências)
    
    Essa apresentação deve durar aproximadamente 30 min.
    
- **ETAPA 2. Análise para Melhoria do Processo Atual e Modelagem BPMN do Processo Melhorado – “TO BE” (Manual e IA)**
    
    O grupo deverá Identificar problemas, gargalos, riscos e/ou oportunidades de melhoria. As melhorias podem estar ligadas à, por exemplo:
    
    - Redução de tempo
    - Redução de custos
    - Diminuição de erros
    - Melhor experiência do usuário
    - Automação
    - Melhoria da comunicação entre áreas
    - entre outros
    
    Após identificação das questões mencionadas acima, o grupo deverá criar o diagrama BPMN incorporando as melhorias propostas de acordo (também) com boas práticas da área escolhida. O modelo TO BE deve utilizar os mesmos elementos do *AS IS*, mas demonstrando claramente:
    
    - Simplificações
    - Automação de tarefas (scripts, serviços)
    - Reestruturação de responsabilidades
    - Redução de retrabalho
    - entre outros
    
    **Entregável:** apresentação descrevendo detalhadamente:
    
    - Análise crítica e identificações de problemas
    - Propostas de melhoria, com base nos problemas e boas práticas da área
    - Diagrama BPMN “TO BE”.
    
    Essa apresentação deve durar aproximadamente 30 min.
